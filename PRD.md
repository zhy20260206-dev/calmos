## 一、产品概要

### 1.1 一句话介绍

**CalmOS** 是一款基于 AI 的轻量级焦虑缓解工具——3 分钟诊断，即时给出具体可执行的缓解方案。

### 1.2 核心数据

| 维度 | 数据 |
|------|------|
| 产品类型 | AI + 心理健康单页 Web 应用，移动端优先 |
| 目标用户 | 20-35 岁有日常焦虑但未达临床诊断标准的人群 |
| 价格策略 | 完全免费（API 成本约 ¥1-2/千次调用） |
| 理论依据 | 认知行为疗法（CBT）+ 接纳承诺疗法（ACT）|
| 在线地址 | https://zhy20260206-dev.github.io/calmos/ |

---

## 二、问题与机会

### 2.1 用户痛点

| 痛点 | 严重程度 | CalmOS 解法 |
|------|---------|------------|
| 日常焦虑普遍但缺乏即时帮助 | 高 | 打开浏览器即可用，无需预约 |
| 心理咨询贵（¥300-800/次）、周期长（1-4 周）| 高 | 完全免费，即时响应 |
| 现有 App 统一推送，不针对具体场景 | 中 | AI 根据 9 题回答生成个性化方案 |
| 不知道从哪入手解决问题 | 中 | 每步指令具体到"现在就做"的动作 |

### 2.2 竞品差距

| 产品 | 短板 |
|------|------|
| Calm / Headspace | 统一推冥想内容，不区分焦虑类型 |
| Woebot | 纯聊天式，输出不够可执行 |
| MindShift | 工具多但缺乏个性化推荐逻辑 |
| Sanvello | 需持续使用，即时效果弱 |

**CalmOS 差异化**：先诊断后开方——9 题识别焦虑类型后精准匹配干预模块，每次只给一个具体动作。

---

## 三、目标

### 3.1 北极星指标

**完成一次完整体验的用户数**（问卷 → 分析 → 干预模块 → 反馈 → 记录保存）。

### 3.2 KPI

| 指标 | 目标 |
|------|------|
| 问卷完成率 | > 80% |
| 干预模块完成率 | > 60%（至少完成 1 个） |
| 焦虑降幅（平均） | ↓ 2 分以上（0-10 分制） |
| 7 日复访率 | > 30% |
| AI 格式异常率 | < 5% |

## 四、功能范围

### 4.1 用户流程

```
入场弹窗 → 主页 → 9题问卷 → 分析过渡 → 结果页 → 干预模块① → 干预模块② → 反馈 → 主页
```

### 4.2 功能清单

| 模块 | 核心功能 | 状态 |
|------|---------|------|
| 入场 | Emoji 头像选择 + 昵称设置 | ✅ |
| 主页 | 时间问候 + 个人标签 + 历史记录 | ✅ |
| 问卷 | 9 题 × 5 条分支路径的智能问卷 | ✅ |
| 分析 | AI 驱动的个性化焦虑诊断 | ✅ |
| 干预 | 6 个 CBT 循证交互模块（两步串联） | ✅ |
| 反馈 | 焦虑分数前后对比 + 帮助程度评估 | ✅ |
| 数据 | 本地 + 云端双写持久化 | ✅ |
| 安全网 | 高危用户自动提示危机热线 | ✅ |

### 4.3 六干预模块

| 模块 | 理论来源 | 解决的问题 |
|------|---------|-----------|
| 行动计划 | CBT 行为激活 | 任务模糊 → 拖延 → 焦虑 |
| 呼吸稳定 | 生理调节 | 焦虑急性生理反应 |
| 不确定性管理 | 暴露与反应预防 | 等待结果时反复预演最坏情况 |
| 对话脚本 | 人际效能训练 | 有话想说但不知道怎么开口 |
| 认知重构 | CBT 三栏表法 | 自我否定的自动化负面想法 |
| 价值锚点 | ACT 接纳承诺疗法 | 说不清的迷茫感和失控感 |

## 五、非功能需求

| 领域 | 要求 |
|------|------|
| 可达性 | 国内用户可正常访问（GitHub Pages + Supabase，绕过 GFW） |
| 离线能力 | 不依赖网络即可查看历史记录和完成基础交互 |
| 隐私 | 不收集 PII，用户通过昵称区分 |
| 安全 | API Key 零前端暴露，数据按用户隔离，RLS 策略保护 |
| 伦理 | 非医疗产品声明 + 危机热线入口 + AI 透明度 |
| 兼容性 | iOS Safari 14+ / Android Chrome 90+ / 桌面主流浏览器 |
