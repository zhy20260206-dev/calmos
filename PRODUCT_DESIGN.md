## 一、产品定义

### 1.1 一句话介绍

**CalmOS** 是一款基于 AI 的轻量级焦虑缓解工具——3 分钟诊断，即时给出具体可执行的缓解方案。

### 1.2 产品类型

AI + 心理健康单页 Web 应用，移动端优先。

### 1.3 目标用户

- 20-35 岁，有日常焦虑但未达到临床诊断标准
- 希望获得即时、私密、低门槛的心理支持
- 对 AI 有基本信任，愿意尝试结构化自助工具

### 1.4 解决的问题

| 痛点 | CalmOS 的解法 |
|------|-------------|
| 焦虑普遍但缺乏即时帮助 | 打开浏览器即可用，无需预约 |
| 心理咨询贵（¥300-800/次）、等待长（1-4 周） | 完全免费，即时响应 |
| 冥想 App 统一推内容，不针对具体场景 | AI 根据个人回答给出针对性方案 |
| 不知道从哪入手 | 每步指令具体到"现在就做"的动作 |

### 1.5 非目标（明确不做）

- 不替代专业心理咨询/精神科诊断
- 不做用户注册/登录系统
- 不做社交/社区功能
- 不收集姓名、手机号等个人身份信息
- 不做长期连续追踪/推送

## 二、核心体验流程

```
入场弹窗 → 主页 → 9题问卷 → 分析动画 → 结果页 → 干预模块① → 干预模块② → 反馈页 → 主页
```

### 2.1 入场弹窗

用户首次访问时，选择一个 emoji 头像 + 输入昵称（2-8 字）。再次访问自动跳过。

**设计意图**：用 emoji 头像降低身份暴露焦虑，昵称而非真名保护隐私。

### 2.2 主页

- 根据时间段显示问候语（早上好/下午好/晚上好）
- 展示用户标签：连续使用次数、累计完成次数、平均焦虑降幅
- 最近 5 条历史记录列表
- 核心 CTA：「开始分析」按钮

**设计意图**：让用户感受到"这是一个属于我的空间"，标签数据提供正向反馈和成就感。

### 2.3 问卷系统（9 题 × 5 条路径）

9 道选择题，约 2-3 分钟完成。Q1 的选择决定后续 Q2 的选项组和 Q4 的深挖方向。

**设计原则**：
1. **最少题目**：9 题收敛到足够做出精准推荐
2. **分支逻辑**：Q1="说不清楚"时跳过 Q2 和 Q4（降至 7 题），不减损体验
3. **给情绪名字**：每道题帮助用户把模糊的焦虑感具象化
4. **降低防御**：emoji 圆形图标 + 彩色左边框替代生硬选项编号

**题目架构**：

| ID | 题目 | 跳过条件 | 选项数 |
|----|------|----------|--------|
| Q1 | 现在让你焦虑的，主要是哪类事？ | 无 | 5 |
| Q2 | 这件事主要是关于？ | Q1=说不清楚 | 5/组（4 组） |
| Q3 | 对你最近的影响有多大？ | 无 | 4 |
| Q4 | 这件事现在卡在哪里？ | Q1=说不清楚 | 4/组 |
| Q5 | 关于这件事，你目前是？ | 无 | 3 |
| Q6 | 焦虑时的身体/情绪状态？ | 无 | 5 |
| Q7 | 思绪主要集中在？ | 无 | 4 |
| Q8 | 现在的你更需要哪种帮助？ | 无 | 4 |
| Q9 | 这件事困扰你多久了？ | 无 | 4 |

**五条路径**：

| Q1 路径 | Q2 情境示例 | Q4 深挖方向 |
|---------|-----------|------------|
| 有件事要做 | 工作汇报/考试/生活事务/健康/拖延事项 | 没思路/没动手/卡住了/担心结果 |
| 在等结果 | 面试/考试/体检/他人态度/审批 | 预想最坏/无力/时间感差/面对 |
| 人际关系 | 伴侣/父母/好友/同事/前任 | 开口/裂痕/看不懂/让对方不舒服 |
| 对自己后悔 | 说过的话/错过机会/无法挽回/没进步/比较 | 回放/否定自己/想修正/担心看法 |
| 说不清楚 |（跳过 Q2、Q4）|（跳过）|

### 2.4 分析过渡页

提交问卷后，展示动画过渡页约 4-7 秒（等待 AI 分析）：
- 中心 blob 变形动画（💫→🌟→🎯→💝 轮换）
- 5 个漂浮小 blob（💭🔍✨💡🧠）
- 打字机效果轮播文案（"正在理解你的情况…"→"识别焦虑来源…"→…）
- 3 个跳动圆点加载器

**设计意图**：AI 调用需要等待，用动画占据注意力防止焦虑加剧。blob 形状呼应品牌语言。

### 2.5 结果页

AI 分析完成后展示：
- **根因标签**：「识别结果」卡片，10 字以内焦虑根因
- **解读摘要**：80-120 字的个性化解读，像朋友复盘
- **evidence 标签**：3 个简短关键词标签
- **2 张模块推荐卡**：标识为第 1 步/第 2 步，每张包含标题、推荐理由、预计时长
- **收尾鼓励语**：20-35 字温暖有力
- 点击任一卡片开始干预，另一张自动排队

**设计意图**：先给出"被理解"的感觉（根因 + 解读），再给出"有方案"的安全感（2 步模块）。

## 三、模块匹配逻辑

### 3.1 核心逻辑

AI 根据 Q1（焦虑类型）和 Q8（需要的帮助）的 5×4=20 种组合，从 6 个预定义干预模块中选取 2 个。匹配表是确定性的，不依赖 AI 自由推理。

### 3.2 20 种匹配表

| Q1 路径 | Q8 先平静 | Q8 理清楚 | Q8 直接行动 | Q8 有人陪 |
|---------|----------|----------|------------|----------|
| 有件事要做 | breathing + action_plan | action_plan + cognitive_reframe | action_plan + cognitive_reframe | action_plan + conversation_script |
| 在等结果 | breathing + uncertainty | uncertainty + cognitive_reframe | uncertainty + action_plan | uncertainty + conversation_script |
| 人际关系 | breathing + conversation | cognitive_reframe + conversation | conversation + cognitive_reframe | conversation + value_anchor |
| 对自己后悔 | breathing + value_anchor | cognitive_reframe + value_anchor | value_anchor + action_plan | cognitive_reframe + conversation |
| 说不清楚 | breathing + value_anchor | value_anchor + cognitive_reframe | value_anchor + action_plan | value_anchor + conversation |

## 四、干预模块设计

### 4.1 理论基础

所有模块基于认知行为疗法（CBT）和接纳承诺疗法（ACT）的循证技术。

| CBT/ACT 技术 | 模块 | 核心机制 |
|-------------|------|---------|
| 行为激活 | 行动计划 | 拆任务至 5 分钟小步，降低启动门槛 |
| 生理调节 | 呼吸稳定 | 4-7-8 呼吸法激活副交感神经系统 |
| 暴露与反应预防 | 不确定性管理 | 三场景分析 + 信息接触限制 |
| 人际效能训练 | 对话脚本 | 双话术脚本化降低开口障碍 |
| 认知重构 | 认知重构 | 事实 vs 加戏三栏表 + 识别认知扭曲 |
| 价值澄清 + 接地 | 价值锚点 | 情绪命名 + 5-4-3 感官接地 |

### 4.2 模块 1：行动计划（action_plan）

- **解决什么问题**：任务模糊 → 拖延 → 焦虑积累
- **给用户什么**：把模糊的大任务拆成 3-4 个"5 分钟就能完成"的小步骤
- **怎么做**：每完成一步点击打勾 ✅，全部勾完激活"完成"按钮
- **时间**：约 5-10 分钟
- **理论依据**：启动是克服拖延的第一障碍，极其简单的第一步（如"打开文档"）足以打破惯性

### 4.3 模块 2：呼吸稳定（breathing）

- **解决什么问题**：焦虑急性生理反应（心跳加速/呼吸急促/胸闷）
- **给用户什么**：4-7-8 呼吸节奏引导（吸 4 秒 → 屏 7 秒 → 呼 8 秒）× 3 循环
- **怎么做**：跟着屏幕上的圆形 blob 动画吸气/呼气，3 次循环自动完成
- **时间**：约 1 分钟
- **理论依据**：激活副交感神经系统，打断焦虑的生理循环

### 4.4 模块 3：不确定性管理（uncertainty_management）

- **解决什么问题**：等待结果时反复预想最坏情况，脑停不下来
- **给用户什么**：三种可能结果的最佳/中间/最坏场景分析 + 等待期可做的具体小事 + 信息接触限制建议
- **怎么做**：横滑浏览三张场景卡片，阅读行动建议
- **时间**：约 3-5 分钟
- **理论依据**："命名即掌控"，把模糊的未知变成有边界的三段式预期；信息接触限制打破反复刷新的强迫行为

### 4.5 模块 4：对话脚本（conversation_script）

- **解决什么问题**：人际焦虑——有话想说但不知道怎么开口
- **给用户什么**：温和版 + 直接版两种开场话术 + 沟通时机和渠道建议 + 48 小时等待窗口建议
- **怎么做**：阅读两种话术，选择适合自己的方式
- **时间**：约 3-5 分钟
- **理论依据**：脚本化降低开口的心理障碍；48 小时窗口防止关系焦虑驱动的反复刷新

### 4.6 模块 5：认知重构（cognitive_reframe）

- **解决什么问题**：自我否定的自动化负面想法
- **给用户什么**：事实 vs 大脑加戏的并排对比 → 识别思维扭曲模式（如"灾难化"、"过度泛化"）→ 更接近真相的替代想法 → "如果是好朋友，你会对 ta 说什么"的温暖视角
- **怎么做**：阅读并排卡片（绿色事实 / 红色加戏），可复制"对好朋友说的话"
- **时间**：约 3-5 分钟
- **理论依据**：三栏表法是 CBT 最经典的工具，外化负面自动化思维

### 4.7 模块 6：价值锚点（value_anchor）

- **解决什么问题**：存在迷茫——说不清的焦虑，感觉失控或空洞
- **给用户什么**：帮情绪取名字 + 今天就能做的一个锚点小行动 + 5-4-3 感官接地练习 + 一句"你可以"的许可语句
- **怎么做**：看到大字号情绪名称 → 读取锚点行动 → 跟着做感官接地（看见 5 个/触摸 4 个/听见 3 个）
- **时间**：约 3-5 分钟
- **理论依据**：ACT 认为不是消除负面情绪，而是带着情绪做有价值的事；感官接地打断思维空转

## 五、反馈与记录

### 5.1 反馈页

- 0-10 滑条自评当前焦虑程度
- 前后分数对比动画（干预前 → 干预后）
- 根据分数变化显示不同鼓励语（"有一点变化，就已经很重要" vs "没关系，找到不适合的方法也是进展"）
- 3 选 1 帮助程度评估（有帮助/一般/没什么帮助）

### 5.2 数据记录

- 每次完成后的记录保存在用户本地
- 主页展示最近 5 条记录：日期、根因、前后分数对比、帮助程度
- 记录帮助用户追踪自己的情绪变化

### 5.3 用户身份

- 首次访问时选择 emoji 头像 + 昵称（2-8 字）
- 昵称用于个人化问候和数据隔离
- 不需要邮箱、手机号、密码
- 保存在本地，下次访问自动识别

## 六、设计语言

### 6.1 核心理念：手绘涂鸦 × 马卡龙

**告别"白大褂"感**：不选严肃/医学风格，用糖果色 + 歪斜卡片传递温暖和安全。

**降低心理防御**：手绘感（微旋转卡片、实色阴影、有机 blob 形状）比完美对齐的 Material Design 更接近"朋友聊天"的氛围。

**一次只做一件事**：每个屏幕只有一个核心操作，不给用户选择负担。

### 6.2 色彩语义

| 颜色 | 色值 | 传达的感觉 | 使用场景 |
|------|------|-----------|---------|
| 奶油黄 | `#FFEAA7` | 温暖、希望 | 用户 blob 头像、高亮标签 |
| 薄荷绿 | `#A8E6CF` | 平静、恢复 | 按钮阴影、进度条、选中态 |
| 深绿 | `#6ECBA4` | 成长、完成 | 步骤完成、"有帮助" |
| 天蓝 | `#A8D8EA` | 信任、清晰 | 备选模块标识、信息卡片 |
| 珊瑚 | `#FFB4A2` | 能量、警示 | 第 1 步标识、高风险提示 |
| 淡紫 | `#C5B4E3` | 温柔、接纳 | 点缀色、认知重构卡片 |
| 暖白 | `#FFFDF9` | 干净、安静 | App 背景色 |
| 深灰 | `#3E3640` | 稳定、可靠 | 文字主色、按钮背景 |

### 6.3 交互原则

1. **每步有进度条**：问卷有进度条，干预模块有 1/2 → 2/2 步骤进度条
2. **所有操作可撤销**：返回按钮无处不在
3. **文案像朋友说话**：用"我们"而非"建议您"，用"你可能在想"而非"你的认知扭曲是"
4. **不给太多选择**：每题 4-5 个选项（符合 Hick's Law）
5. **点击反馈即时**：< 150ms 视觉反馈
6. **焦虑感不叠加**：等待动画让 AI 调用时间变得可接受

### 6.4 排版与字体

- 字体：系统原生中文字体栈（PingFang SC / Microsoft YaHei / Noto Sans SC）
- 不依赖 Google Fonts（避免国内加载失败）
- 标题 900 字重（22px/19px），正文 400-600 字重（16px，1.6 行高）
- 卡片圆角 12-16px，微旋转 -0.2° ~ -0.5°
- 2.5px 实色描边 + 低透明度阴影，维持手绘感

## 七、伦理与安全边界

### 7.1 非医疗声明

- CalmOS 不是医疗产品，不替代专业心理咨询、精神科诊断或药物治疗
- 使用 CalmOS 不构成任何形式的治疗关系
- 产品在用户界面中明确提示 AI 分析性质

### 7.2 危机响应

对于自评高焦虑（Q3 选"影响非常大"且 Q9 选"很长一段时间"）的用户：
- 展示危机热线弹窗：全国心理援助 400-161-9995 / 北京心理危机研究 010-82951332
- 提供返回主页按钮

### 7.3 数据隐私

- 不收集姓名、手机号、身份证、IP 地址等个人身份信息
- 用户通过昵称区分，不要求真实姓名
- 数据保存在用户本地设备和自愿同步的云端
- 无第三方数据共享，无广告追踪

### 7.4 AI 透明度

- 用户知晓分析由 AI 生成（产品名 CalmOS 暗示工具/操作系统角色）
- 匹配表是确定性的（查表而非自由推理），确保推荐的可预测性
- 分析结果经完整性验证后才展示，异常时提示重试
